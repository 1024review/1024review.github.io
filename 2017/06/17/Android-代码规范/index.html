Android 代码规范

2017-06-17

1.类名禁止出现特殊符号

2.变量名不能以特殊符号开头或结尾

3.变量名中禁止使用拼音加英文，更不能出现汉字。应该使用全英文

4.类名命名必须使用大驼峰命名法，变量必须使用小驼峰命名法。静态变量全部大写，特殊单词（领域模型的相关命名）例外，例如DO / BO / DTO / VO 等

5.见名知意，命名简单明了

6.缩写单词要参照习惯，不能主观更改，例如AbstractClase写成AbsClass

7.抽象类命名使用 Abstract 或 Base 开头；异常类命名使用 Exception 结尾；测试类
命名以它要测试的类的名称开始，以 Test 结尾。项目上线前统一删除Test类

8.命名中避免关键字

9.包名统一小写，com.domain.模块.作用 ，Model类和数据接口类单独列包

10.枚举类和变量以Enum（enum）开头（结尾）

11.类名命名为 类型（Enum/Abstract）+ 类名（GetMessage）+ 定义（Utils/Exception）；变量命名可以单独说明在本类中的作用，例如userMessages；id的命名应该为 “模块_类型_变量名“,个别需要加标识符。
